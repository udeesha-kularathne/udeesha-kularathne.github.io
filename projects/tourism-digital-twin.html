Tourism Digital Twin & Demand Forecasting System

A mobile-first AI system that combines hybrid time-series forecasting, scenario simulation, and interactive geospatial visualization to model tourism demand, safety, traffic, and crisis impact at both district and national levels.

Role AI Engineer & Full-Stack Developer
Timeline 2024 – 2025
Category AI/ML, Mobile App, Decision Support

Project Overview

This project was designed as a practical tourism decision-support system rather than a standalone forecasting model. The goal was to create a “digital twin” that mirrors how tourism demand evolves over time and reacts to external factors such as seasonality, weather conditions, public sentiment, and crisis indicators.

The system integrates a hybrid machine learning forecasting pipeline with a mobile application built using React Native (Expo). Users can explore live district-level conditions through an interactive map and simulate future scenarios using a configurable what-if engine.

The backend exposes structured APIs for forecasts, simulations, and historical summaries, enabling clean separation between ML logic, data services, and frontend visualization. The architecture is designed to scale from exploratory analytics to operational planning tools without structural changes.

Note: Dataset sources, exact feature thresholds, and operational tuning parameters are abstracted to preserve commercial applicability and confidentiality.

Key Features

Hybrid Time-Series Forecasting

Combines tree-based models and deep learning (LSTM) into a weighted ensemble optimized for time-series prediction accuracy and stability.

Automatic Feature Engineering & PCA

Applies lag features, rolling averages, seasonal indicators, and automatic dimensionality reduction to stabilize multivariate inputs.

Interactive Digital Twin Map

Visualizes district-level demand, traffic, safety, and sentiment through animated geospatial overlays and live markers.

What-If Scenario Simulator

Allows users to simulate changes in climate, crisis levels, and seasonality to observe projected impacts on tourism, safety, and congestion.

Results & Engineering Impact

The hybrid forecasting strategy significantly improves reliability over single-model approaches. Tree-based models capture non-linear interactions between external drivers such as seasonality and crisis indicators, while the LSTM component models temporal dependencies across historical demand trends. The ensemble weighting is optimized for percentage-based error, producing stable forecasts suitable for planning scenarios.

From an ML engineering perspective, the system demonstrates disciplined time-series handling: strict temporal splits, lag-aware feature construction, and dimensionality reduction applied only to relevant feature groups. This minimizes data leakage, improves generalization, and keeps inference behavior aligned with training conditions.

On the backend, the project follows clean service-oriented principles. Forecasting, simulation logic, and aggregation are exposed through predictable APIs, enabling the frontend to remain fully decoupled from model internals. This separation allows models to be retrained or replaced without breaking mobile clients or visualization logic.

The frontend architecture emphasizes usability and interpretability rather than raw prediction display. Interactive maps, animated indicators, and scenario sliders are designed to translate complex ML outputs into intuitive signals that non-technical stakeholders can explore confidently.

Overall, the project demonstrates strong software engineering fundamentals alongside applied machine learning: modular design, reproducible pipelines, scalable APIs, and a product-oriented mindset. The result is a system that behaves like a real decision-support product rather than a research prototype.
