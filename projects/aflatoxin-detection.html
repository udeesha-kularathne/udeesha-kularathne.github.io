Aflatoxin Detection & Yield Prediction System

A full-stack ML decision-support system for agriculture that predicts aflatoxin risk and crop yield, provides geo-aware insights, and delivers transparent explanations through feature importance and interpretable model outputs.

Role Full-Stack Developer & ML Engineer
Timeline 2023 - 2024
Category AI/ML, Web Application

Project Overview

This project was built to convert complex agriculture datasets into practical, decision-ready outputs. The system combines predictive modeling with a dashboard-driven interface so that users can assess contamination risk, estimate yield outcomes, and understand the key environmental or management factors influencing those predictions.

The solution follows an end-to-end workflow: raw datasets are cleaned and standardized, location-related fields are normalized into consistent geographic groups, and a feature engineering layer prepares inputs for both regression and classification tasks. The ML layer supports multiple model families and evaluation workflows, while explainability tooling (feature importance + local/global interpretations) ensures outputs remain transparent and defensible.

The frontend is designed as a data-heavy analytics interface (EDA, prediction forms, geo summaries, safe-range guidance, and feature importance exploration). The backend exposes structured APIs for data and ML services, keeping the UI decoupled from model internals and supporting long-term maintainability.

Note: Specific datasets, thresholds, and proprietary optimization details are intentionally abstracted to respect commercial and confidentiality constraints.

Key Features

Aflatoxin Risk Classification

Predicts contamination risk as a classification task using structured preprocessing pipelines and ensemble modeling, returning both decisions and confidence-oriented outputs.

Yield Prediction (Regression)

Estimates expected yield using regression models optimized for tabular data, enabling scenario comparisons across different environmental and management conditions.

Time & Location Normalization

Includes robust preprocessing to standardize inconsistent field inputs (such as location naming), enabling reliable aggregation, geo summaries, and stable model inputs.

Explainable AI & Feature Importance

Provides multi-method explainability (linear coefficients, tree importances, permutation tests, SHAP, and local explanations) to justify predictions and improve trust.

Safe-Range Guidance

Computes practical “safe/adjustable” ranges for key inputs to support decision-making, enabling users to explore changes that may reduce risk or improve outcomes.

Analytics Dashboard & EDA

A React-based analytics interface for exploratory data analysis, prediction workflows, geo averages, and interpretability outputs—structured for clarity and fast navigation.

Results & Engineering Impact

The main value of this system is the end-to-end integration of predictive modeling with a usable, decision-support interface. The workflow is not limited to “training a model”—it includes robust preprocessing, repeatable evaluation, explainability outputs, and a UI that makes analysis accessible to non-technical users while still remaining detailed enough for technical review.

The ML layer demonstrates a practical blend of model families for tabular prediction problems, including ensemble learning strategies and pipeline-based preprocessing. This design reduces risk of inconsistent transformations, supports reproducibility, and keeps inference aligned with training-time logic. Explainability tooling adds transparency by validating which factors drive risk and yield outputs, supporting trust and accountability.

From an engineering standpoint, the project emphasizes clear separation of concerns: UI components handle exploration and visualization, while backend services expose stable APIs and isolate ML logic. This structure supports maintainability, makes the system easier to extend with additional analytics modules, and keeps model changes decoupled from interface development.
