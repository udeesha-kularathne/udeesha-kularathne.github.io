BabyCry — Infant Cry Reason & Adaptive Soothing System

An edge-first smart baby monitoring system that detects crying, classifies likely reasons, generates practical caregiver advice, and triggers emotion-aware lullaby playback through an ESP32-based IoT audio device — designed for low latency, offline operation, and real-time feedback.

Role Full-Stack / Edge ML & IoT Engineer
Timeline 2023 - 2024
Category AI/ML, IoT, Mobile Application

Project Overview

BabyCry was built to turn raw audio signals into actionable caregiver support. Instead of only detecting “cry vs. not cry”, the system follows a multi-step workflow: it detects crying events, estimates the most likely reason category, generates short advice guidance, and selects an appropriate soothing audio track that can be played immediately through an IoT speaker device.

The solution is designed as a practical edge system. The mobile app runs real-time inference using compact TensorFlow Lite models and a consistent audio preprocessing pipeline (log-mel based features). When a crying event is detected, the app logs events, shows confidence-oriented outputs, and triggers the IoT playback layer via lightweight network commands — enabling low-latency soothing without requiring cloud dependence.

From an engineering perspective, the project emphasizes real-time audio handling, modular ML inference, and resilient device communication. The audio streaming path includes sequence-aware packet handling and jitter buffering, while the app structure uses clean separation between UI, services, inference utilities, and storage.

Note: Model weights, training datasets, and certain tuning thresholds are intentionally abstracted to respect commercial and confidentiality constraints.

Key Features

Two-Stage Cry Understanding

A staged inference pipeline: Stage 1 detects crying vs background, and Stage 2 classifies the likely crying reason category — keeping the system fast and reducing false triggers.

Real-Time Advice Generator

Converts model outputs into short caregiver guidance (what to try first, what to check next), designed to be understandable and actionable in the moment.

Emotion-Aware Lullaby Playback

A music selection layer that recommends calming audio tracks based on the detected reason/emotion profile, enabling adaptive soothing rather than one fixed track.

Mobile App + Background Service

A mobile-first experience with a continuous detection service, notifications, event logging, and a dashboard UI — built for reliability during long-running sessions.

Robust ESP32 Device Communication

Lightweight network control for the ESP32 audio device plus resilient streaming behavior (sequence-aware packets, jitter buffering, and graceful loss handling).

Edge Model Optimization

Models are optimized for mobile/edge deployment using compression techniques such as quantization, and training strategies like distillation to balance accuracy with real-time performance.

Results & Engineering Impact

BabyCry demonstrates a practical “edge AI + IoT” architecture where inference, guidance, and intervention happen in real time. By using a staged model design and optimized TFLite deployment, the system can respond quickly to crying events and reduce unnecessary triggers — which is critical in a continuous monitoring scenario.

The project also highlights engineering depth beyond modeling: resilient audio streaming design, sequence-aware packet handling, jitter buffering, and a clean separation between UI, background detection, inference utilities, and device control. This structure makes it easier to expand the system with additional classes, improved advice logic, or alternative playback devices without reworking core components.

On the ML side, the pipeline focuses on edge readiness: consistent audio preprocessing, evaluation workflows, and model compression strategies (distillation + quantization) to achieve a balanced tradeoff between accuracy, latency, and mobile resource constraints.
